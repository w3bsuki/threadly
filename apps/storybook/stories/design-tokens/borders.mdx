## Usage Guidelines

### When to Use Borders

1. **Separation**: Define boundaries between distinct content areas
2. **Emphasis**: Highlight important elements or states
3. **Grouping**: Unite related elements within a border
4. **Interaction**: Indicate interactive elements or focus states

### Border Best Practices

<div className="grid grid-cols-1 md:grid-cols-2 gap-6 my-8">
  <div className="p-4 bg-muted/50 rounded-lg">
    <h4 className="font-semibold mb-2">✅ Do</h4>
    <ul className="space-y-1 text-sm">
      <li>• Use consistent border widths across similar elements</li>
      <li>• Match border radius to component hierarchy</li>
      <li>• Ensure sufficient contrast for visibility</li>
      <li>• Use semantic color tokens for borders</li>
      <li>• Consider removing borders in dark mode for cleaner look</li>
    </ul>
  </div>
  
  <div className="p-4 bg-muted/50 rounded-lg">
    <h4 className="font-semibold mb-2">❌ Don't</h4>
    <ul className="space-y-1 text-sm">
      <li>• Mix different border styles in the same component group</li>
      <li>• Use borders as the only indicator of state</li>
      <li>• Create borders lighter than the background</li>
      <li>• Overuse borders - embrace whitespace</li>
      <li>• Use arbitrary border values outside the scale</li>
    </ul>
  </div>
</div>

### Accessibility Considerations

- **Contrast**: Ensure borders meet 3:1 contrast ratio against backgrounds
- **Focus Indicators**: Use distinct border treatments for focus states
- **Not Sole Indicator**: Don't rely only on borders to convey meaning
- **Consistent Patterns**: Use predictable border patterns for similar elements

### Responsive Borders

Consider adjusting borders for different screen sizes:

```css
/* Mobile - lighter borders */
@media (max-width: 768px) {
  .card {
    border-width: var(--border-width-1);
  }
}

/* Desktop - standard borders */
@media (min-width: 769px) {
  .card {
    border-width: var(--border-width-2);
  }
}
```
